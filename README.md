# Task Manager

A simple full-stack task management app. Built as part of a Full Stack Developer coding assessment.

Live demo: https://task-manager-ebon-chi-19.vercel.app/
Backend API: https://task-manager-api-hxr2.onrender.com/api/tasks

## Features

- Add, edit, delete tasks
- Mark tasks as complete
- Current weather widget (OpenWeather API integration)

## Tech Stack

**Frontend:** React (Vite), Axios, CSS
**Backend:** Node.js, Express
**Database:** MongoDB Atlas (Mongoose)
**Deployment:** Vercel (frontend), Render (backend)

## Project Structure

## Setup Instructions

### Prerequisites

- Node.js (v18+)
- A MongoDB Atlas account (free tier works fine)
- An OpenWeather API key (free, from openweathermap.org)

### 1. Clone the repo

```bash
git clone https://github.com/AhubHashmi/task-manager.git
cd task-manager
```

### 2. Backend setup

```bash
cd server
npm install
```

Create a `.env` file inside `server/` with:

```
PORT=5000
MONGO_URI=your_mongodb_atlas_connection_string
```

Run the backend:

```bash
npm run dev
```

Server runs on `http://localhost:5000`.

### 3. Frontend setup

Open a new terminal:

```bash
cd client
npm install
```

Create a `.env` file inside `client/` with:

```
VITE_API_URL=http://localhost:5000/api/tasks
VITE_WEATHER_API_KEY=your_openweather_api_key
```

Run the frontend:

```bash
npm run dev
```

App runs on `http://localhost:5173`.

## Database Schema

**Task Model** (MongoDB collection: `tasks`)

| Field | Type | Notes |
|---|---|---|
| `_id` | ObjectId | Auto-generated by MongoDB |
| `title` | String | Required |
| `description` | String | Optional, defaults to empty string |
| `completed` | Boolean | Defaults to `false` |
| `createdAt` | Date | Auto-generated (timestamps) |
| `updatedAt` | Date | Auto-generated (timestamps) |

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/tasks` | Get all tasks |
| POST | `/api/tasks` | Create a new task |
| PUT | `/api/tasks/:id` | Update a task (edit or mark complete) |
| DELETE | `/api/tasks/:id` | Delete a task |

## Notes

- Render's free tier spins down after inactivity, so the first API request after idle time may take 30-50 seconds to respond (cold start).
- CORS is currently open for development; in a production setting this would be restricted to the deployed frontend's domain.
